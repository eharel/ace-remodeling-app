fix: preserve component selection when editing project/component descriptions

## Problem
When editing a project or component description, the selected component would reset to either:
- The first component in the project (if no URL param)
- The component specified in the URL param (if present)

This happened because the `useProjectComponentSelection` hook was re-running when the projects array updated after an edit, and it would reset the selection based on the URL param even though the param hadn't changed.

## Solution
- Track previous `componentIdParam` value to detect when it actually changes
- Only respect URL param when it changes (initial load or navigation)
- Preserve current selection when projects array updates but param is unchanged
- Use ref to track current selection across project updates

## Changes
- Modified `useProjectComponentSelection` hook to track param changes
- Preserve selection when projects update but param hasn't changed
- Fix `updateComponent` to use updated state instead of stale closure value

## Impact
Users can now edit descriptions without losing their place - the selected component remains visible after saving changes.
